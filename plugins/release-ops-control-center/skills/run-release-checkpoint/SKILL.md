# Run Release Checkpoint

Use this skill when a launch or staged rollout needs a quick release-ops pass.

## When to Use

- A release is about to start or is already in progress
- The team needs a concise launch checkpoint
- Rollout metrics, owners, or stop conditions need review

## Instructions

1. Identify the current rollout phase.
2. Confirm owners, health metrics, and rollback triggers.
3. Use the MCP server only for release-oriented context gathering.
4. End with the top launch risks and next checkpoint.
